# Spectrotyper: Server

The `@spectrotyper/server` package is the backend for the Spectrotyper application, built with Express.js. It converts text into frequency-based WAV files.

## Technologies

- **Server**: Express.js
- **Audio Encoding**: `node-wav`
- **Text Rendering**: `js-pixel-fonts`
- **Development Tool**: `tsx` for hot-reloading and environment variable management.

## Getting Started

### Development

Run the server in development mode:

```bash
npm run dev
```

The server will be available at [http://localhost:3001](http://localhost:3001).

### Build

Build the server for production:

```bash
npm run build
```

### Other Commands

- `npm run start`: Run the built server from `dist`.
- `npm run typecheck`: Run TypeScript type-checking.
- `npm run format`: Format code using Prettier.

## API Endpoints

### `GET /health`

Checks the status of the server.

- **Response**: `{"status": "ok"}`

### `POST /convert`

Converts a string of text into a WAV audio file.

- **Request Body**: `{"text": "string"}` (max 200 characters)
- **Response**: A binary WAV file (`audio/wav`).

## Synthesis Logic

1. **Text as Pixels**: Text is rendered as a 7-row grid of pixels.
2. **Frequency Bands**: Each row represents a frequency from 1000Hz to 3000Hz.
3. **Column Summation**: For each column of pixels, a composite sine wave is generated by summing the frequencies corresponding to the "lit" pixels.
4. **Windowing**: A Hann window is applied to each column's audio segment to prevent clicks between adjacent segments.
5. **Normalization**: The resulting PCM data is normalized to ensure maximum volume without clipping.
